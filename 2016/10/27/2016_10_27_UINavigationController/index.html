extendedLayout、automaticallyAdjustsScrollViewInsets、extendedLayoutIncludesOpaqueBars

iOS 7 以后在 ViewController 里面引进了一系列属性用于管理页面布局。下面三个属性是 Apple 官方提供的文档解释：

  • edgesForExtendedLayout：The extended edges to use for the layout.

  • automaticallyAdjustsScrollViewInsets：A Boolean value that indicates whether the view controller should automatically adjust its scroll view insets.

  • extendedLayoutIncludesOpaqueBars：A Boolean value indicating whether or not the extended layout includes opaque bars.

iOS 7 以后，ViewController 的 View 的布局默认是全屏的，NavigationBar 默认是半透明的。

edgesForExtendedLayout

将 edgesForExtendedLayout 设置成 UIRectEdgeNone，表明 View 是不要扩展到整个屏幕的（但 NavigationBar 依然是半透明的），而是 View 是从 NavigationBar 的下方开始算。

automaticallyAdjustsScrollViewInsets

如果页面是 UIScrollView 或者 UITableView，通常我们希望 UIScrollView 或者 UITableView 的内容显示在 navigationBar 下面。通过设置edgesForExtendedLayout = UIRectEdgeNone或者self.navigationController.navigationBar.translucent = NO;可以让 view 的布局从 navigationBar 下面开始，不过它有一个副作用就是当页面滑动的时候，view 是没有办法占满全屏的。

navigationController.navigationBar.translucent

navigationController.navigationBar.translucent = YES;用于设置 navigationBar 不透明，且 view 是从 navigationBar 下方开始的，即 view 不能占满整个屏幕，即 view 的 (0, 0) 位于 navigationBar 下方（同 iOS 7 之前的特性）。

extendedLayoutIncludesOpaqueBars

不管 navigationController.navigationBar.translucent 的值是 true 还是 false，self.extendedLayoutIncludesOpaqueBars = YES 都会使得 view 是从 navigationBar 下方开始算起的。
如果状态栏是不透明的，那么页面的布局默认是不会包含状态栏的，除非将这个属性设置成为YES。所以如果你的页面扩展到Navigation Bar (edgesForExtendedLayout=UIRectEdgeAll)，要是这个属性设置成NO (default), 如果状态栏是不透明的话，页面是不会扩展到状态栏的。

automaticallyAdjustsScrollViewInsets 并不能帮助我们正常计算ScrollView/TableView的Inset，这时候就自己设置咯。

总结

情景：self.window.rootViewController 是 UINavigationController，UINavigationController 的 rootViewController 是 UIViewController， UIViewController 有个 UITableView 属性，UITableView 的 frame 为 UIViewController 的 view 的 bounds。

  • 默认

    controller.view 占满整个屏幕，navigationBar 是默认的半透明；

  • self.edgesForExtendedLayout = UIRectEdgeNone;

    navigationBar 依然是默认的半透明，但 controller.view 从 navigationBar 下方开始算；
    设置了 translucent = false 后，变为不透明，但是 view 依然不占满整个屏幕；

  • self.automaticallyAdjustsScrollViewInsets = YES;

    navigationBar 依然是默认的半透明，controller.view 占满整个屏幕，同默认。
    但设置了 translucent = false 后，变为不透明，但是 view 也不占满整个屏幕；

  • self.extendedLayoutIncludesOpaqueBars = YES

    navigationBar 依然是默认的半透明，controller.view 占满整个屏幕，同默认。
    但设置了 translucent = false 后，变为不透明，虽然 controller.view 依然占满整个屏幕，但是 controller.view 上的 tableView 从 navigationBar 下方开始计算（好奇怪）；

  • navigationController.navigationBar.translucent = false;

    navigationBar 变为不透明，controller.view 从 navigationBar 下方开始算；

translucent = false（不透明） translucent = ture（半透明） 结果
默认 不占满 占满
self.edgesForExtendedLayout = UIRectEdgeNone 不占满 不占满（黑背景） 奇怪
self.automaticallyAdjustsScrollViewInsets = YES 不占满 占满 同默认
self.extendedLayoutIncludesOpaqueBars = YES 占满 占满 特别

结论：

  1. 不管 navigationBar 是半透明还是不透明，都希望 controller.view 能占满屏幕，因为这样风格统一，方便计算，且不影响以后的扩展，且不影响加在上面的 UIScrollView 或者 UITableView 的内容显示，所以用self.extendedLayoutIncludesOpaqueBars = YES最好了；
