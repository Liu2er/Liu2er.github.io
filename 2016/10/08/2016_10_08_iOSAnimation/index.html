iOS 图层

frame、bounds、position、center

视图（UIView）有三种比较重要的布局属性：frame，bounds，center，图层（CALayer）对应叫做：frame，bounds，position。position 和 center 表示一个意思。

  • bounds 和 position 是存储性属性
  • frame 是计算性属性（虚拟属性），它的值根据 bounds、position 和 transform（ CALayer 的）计算而来。

UIView 和 CALayer 都有 transform 属性，但是 UIView 的 transform 是 CATransform3D 类型，而 CALayer 的 transform 是 CGAffineTransform 类型。

当视图发生仿射变换时，比如平移、旋转、缩放，经验证发现：position 和 bounds 都不会改变，而 frame 会改变。并且当对图层进行缩放时，frame.size 会和 bounds.size 不一致。

Core Animation

相比较于 Quartz 2D 绘图，使用 Core Animation 创建动画不仅简单，而且性能更好，原因如下：

  • Core Animation 动画在单独线程中完成，不会阻塞主线程；
  • Core Animation 动画只会重绘界面上变化的部分（局部刷新）。

Core Animation 的核心是 CALayer。Core Animation 主要涉及的 API 有：CAAnimation、CATransition、CAPropertyAnimation、CABasicAnimation、CAKeyframeAnimation、CAAnimationGroup。
